# Task 05: Гарантиране на стабилността на резултата чрез структуриран изход (JSON)

**Име на задачата:** 
Преминаване от суров Markdown към структуриран JSON отговор от AI модела.

**Цел:** 
Да се подсигури, че AI моделът винаги връща резултата в точната структура (6 отделни секции), чрез използване на JSON формат, за да се избегне "счупване" на потребителския интерфейс.

**Контекст:** 
В Task 02 реализирахме връзката с AI, разчитайки моделът да върне един дълъг Markdown текст, който ние да разделим на секции. Това е крехко (fragile) – ако AI промени малко заглавие, интерфейсът може да не го разпознае. За професионален инструмент трябва резултатът да е детерминистичен: обект с точно 6 полета.

**Връзка с продукта:** 
Елиминира един от основните технически рискове при AI приложенията (непостоянно форматиране), правейки продукта надежден за крайния потребител и отговаряйки на критерия за "структуриран и лесен за разбиране резултат".

**Какво влиза в задачата:**
- Промяна на системния промпт (System Prompt), изискваща от AI да върне отговора като валиден JSON обект със следните ключове: `summary`, `problems`, `recommendations`, `structure`, `priorities`, `next_steps`.
- Настройване на AI API заявката да поддържа JSON режим (напр. `response_format: { type: "json_object" }` при OpenAI).
- Обновяване на JavaScript логиката да парсва получения JSON (`JSON.parse()`) вместо да използва сложни регулярни изрази (Regex) за цепене на Markdown.
- Директно подаване на всяко поле от JSON обекта към съответния HTML контейнер на екрана.

**Какво не влиза в задачата:**
- Промяна на съдържанието или логиката на `skill.md` за самия анализ.
- Промени по визуалния дизайн (CSS) на резултатите.

**Очакван резултат:** 
Кодът става много по-чист и устойчив. Независимо какво съдържание генерира AI, то винаги ще бъде разпределено перфектно в 6-те обособени кутии на екрана.

**Критерии за приемане:**
- AI винаги връща валиден JSON.
- Приложението успешно разчита JSON отговора и попълва UI секциите.
- Ако AI случайно върне невалиден JSON, приложението хваща грешката (try/catch) и показва съобщение: "Възникна грешка при форматирането на анализа. Моля, опитайте отново."

**Рискове или неясноти:**
- Възможно е AI моделът да вкара Markdown синтаксис вътре в самите JSON стойности (напр. `**bold**` текст вътре в полето `problems`). UI логиката трябва да продължи да поддържа рендериране на базов Markdown за всяко отделно поле.
